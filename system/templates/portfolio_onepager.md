# 포트폴리오 1페이지 템플릿 — 프로젝트별

> 채용담당자가 1분 안에 "무엇을 / 어떻게 / 결과"를 파악하게 만든다.
> 톤: 1인칭, 채용담당자 직격, 제공자 태도. 회고·메타분석 톤 금지.
> 숫자 규칙: **측정한 수치만 적는다. 가짜 KPI 금지.** 측정 안 한 결과는 정성으로 쓰거나 빈칸.

---

## [프로젝트명] — 한 줄 정의 (Role: ______)

**맨 위 큰 숫자 2개** (측정한 것만, 없으면 이 줄 삭제):
> `KPI1: ___ → ___` ｜ `KPI2: ___ → ___`

### 문제 (1문장, 가능하면 정량)
누구의 어떤 문제. 막연어 금지 — "○○ 사용자가 △△ 때문에 □□ 못 함".

### 가설 / 목표
무엇을 증명하려 했나. 목표가 숫자면 숫자로.

### 내가 한 것 (3줄)
1. (기획·가설) ___
2. (AI 워크플로우 설계 — 무엇을 자동화/오케스트레이션했나) ___
3. (검증 — 어떻게 측정/실험했나) ___

### 결과
- 정량(측정한 것만): ___
- 정성(숫자 없을 때): "___를 통해 ___를 확인" 식. 과장 금지.
- 증거 링크: 데모영상(60초) / 스크린샷 / 리포트 → ___

### 배운 점 (1줄) + 다음 액션 (1줄)
- 배운 점: ___
- 다음: ___

### 작성 체크 (게시 전)
- [ ] 적힌 숫자 전부 실제 측정값인가? (추정·예시값 0)
- [ ] 첫 10초/첫 줄에 핵심 가치 노출되나?
- [ ] "AI를 도구로 썼다"가 아니라 "AI 워크플로우를 설계했다"로 읽히나? (이직 타겟: AI Workflow Architect)
