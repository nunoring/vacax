점검 범위: $ARGUMENTS (없으면 전체)

너는 VacAX 시스템 감사자다. 아래 체크리스트를 순서대로 실행하고 이슈만 보고한다. 정상 항목은 생략.

## 점검 항목

### A. CLAUDE.md 상태
- 파일 크기 확인 (줄 수 카운트)
- 오래된 날짜 참조 있는지 (현재 날짜 기준 3개월 초과)
- 중복 규칙 존재 여부 (같은 내용 2곳 이상)
- 직군 시스템 정의가 실제 에이전트 타입과 일치하는지

### B. 메모리 상태
- `~/.claude/projects/C--Users-vacma-vacax/memory/` 파일 목록
- MEMORY.md 인덱스와 실제 파일 불일치 확인
- 3개월 이상 된 stale 메모리 있는지 (파일명·내용 기준)

### C. 스킬 생태계
- 설치된 플러그인 목록 (`claude plugin list`)
- 비활성화되거나 중복 기능 스킬 있는지
- `.claude/commands/` 커스텀 명령어 목록

### D. 운영 문서 정합성
- `active/` 폴더 하위 프로젝트별 4개 파일(STATE/NEXT/DECISIONS/LOG) 존재 여부
- STATE.md 마지막 수정일이 7일 이상 지났으면 stale 플래그
- NEXT.md에 기한 없는 액션 있는지

### E. GitHub 보안
- `active/` 폴더에 `.gitignore` 미커버 민감 파일 있는지 (config.dart, .env, API_key 등)
- 최근 3개 commit에 민감 키워드 포함 여부 확인 (`git log -3 --stat`)

### F. Notion 연결 상태
- 인사이트 라이브러리 안티패턴/검증패턴 페이지 접근 가능한지 fetch 확인
- 최근 세션 인수인계 페이지 업데이트 날짜 확인

## 실행 순서

1. A~E 항목 순서대로 파일 Read/Bash 실행
2. F 항목 Notion fetch
3. 이슈 목록 작성

## 출력 형식

```
## 🔍 VacAX 시스템 점검 결과
점검 일시: [오늘 날짜]

### 🚨 즉시 조치 필요
- [항목]: [이슈 1줄] → 권장 액션

### ⚠️ 주의 (이번 주 내)
- [항목]: [이슈 1줄] → 권장 액션

### 📌 보류 (다음 점검 시)
- [항목]: [이슈 1줄]

### ✅ 정상
[정상 항목 수]개 항목 이상 없음

---
다음 점검 권장일: [오늘 + 7일]
```

이슈 없으면 "전체 정상" 한 줄로 끝낸다.
