## 1. 6상자 카피라이팅 (제목·헤드·한 줄 카피)

작성한 제목이 아래 6상자 중 **최소 3개** 통과 필수. 통과 못 하면 다시 작성.

### 6상자 정의 + 어휘 예시

| # | 상자 | 핵심 | 어휘 예시 |
|---|---|---|---|
| 1 | **욕구성** | 돈/건강/관계와 직결 | "돈 버는", "취업되는", "건강 유지", "여자/남자 만나는" |
| 2 | **날먹성** | 날로 먹는 기분 | "쉽고 간단한", "그대로 따라하는", "한 번 알면 평생", "공짜로" |
| 3 | **충격성** | 몰입도 계단식 상승 | 의외성(반대 결과) / 확신성(파격적 숫자) / 도발성(반발 자극) / 위험성(공포·불안) |
| 4 | **희소성** | 귀하게 만들기 | "퍼지면 큰일", "아무도 모르는", "한정", "참신한", "남들은 모르는" |
| 5 | **특정성** | 생각의 양 줄여줌 | 구체 숫자(3가지, 5단계) / "이것" / "공식·법칙·루틴" |
| 6 | **정체불명** | 낯선 명칭 (단독 X) | 메소드·이론·기술에 자체 작명. 매력 문구와 동반 필수 |

### 통과 예시 (글쓰기 사업 카피)

```
원문:              글쓰기
+욕구성:           글쓰기로 돈 버는 법
+날먹성:           글 몇 줄 그대로 따라 써서 돈 버는 법
+충격성:           머리 나쁠수록 더 유리한 글 몇 줄로 경제적 자유 얻는 법
+희소성:           머리 좋으면 절대 모르는 글 몇 줄로 경제적 자유 얻는 비밀
+특정성:           머리 나쁜데 월 1억 버는 사람들만 몰래 쓰는 글쓰기 루틴 2가지
+정체불명:         머리 나쁜데 월 1억 버는 사람들만 몰래 쓰는 글쓰기 기술 '조립 루틴'
```

### 정체불명 상자의 부가 효과
독자적 명칭(이론·기술·메서드명) 누적 → 세상에 없는 정체성 → **가격 결정권 확보** (가격경쟁 회피).

VacAX에서 만들 작명 예: "AX 기획 N그래프", "AI 멀티 에이전트 스튜디오", "박카스 운영 시스템" 등.

## 2. 뇌과학 글공식 6단계 (모든 콘텐츠 구조)

모든 외부 노출 글·영상·랜딩은 아래 6단계 구조 통과.

| 단계 | 목적 | 분량 | 중요도 |
|---|---|---|---|
| 1. **메인 후킹** | "어, 이게 뭐지?" | 5% | ★★★★★ 가장 중요 |
| 2. **서브 후킹** | "왜 계속 봐야 하지?" | 중간 | ★★★★ |
| 3. **효과 입증** | "와 진짜 되네!" | 중간 | ★★★★ |
| 4. **전문가 포지셔닝** | "오 진짜 전문가네!" | 90% (본론) | ★★ 가장 낮음 |
| 5. **콜투액션** | "이것만 하면 된다?" | 짧음 | ★★★★ |
| 6. **마무리 후킹** | "무조건 해야겠는걸!" | 짧음 | ★★★★★ 정점·종점 원칙 |

### 1. 메인 후킹 (5%, 사업 성패 결정)
- 썸네일·제목·첫 줄·슬로건·브랜드명
- **6상자 카피** 적용 필수

### 2. 서브 후킹 (집중 유지)
- **고객이 얻을 이득 명시** (계속 볼 이유)
- **엥커링**: 소품·BGM·말투·캐릭터로 감정 형성 → 기억 새김
- 인간은 **언어가 아닌 감정**으로 기억
- 부정 감정이라도 강렬하면 OK (기억에 남는 게 우선)

### 3. 효과 입증 (직관 증거)
- 장황한 설명 X → **시각·직관 자료 O**
- 본인 성과 없으면 **외부 성과 레버리지** (뉴스 사례·유명인·지인·차트)
- 단 조건: 원인·기술 분석 + 가치 공유 가능해야 (아니면 사기)

### 4. 전문가 포지셔닝 (본론, 분량 가장 많지만 중요도 가장 낮음)
- 99% 사람은 본론 잊음
- 본론 유일 목적: **"나는 전문가다"** 메시지 전달
- 정성껏 채워야 하는 이유:
  1. 효과 입증이 운 X, 인과관계 O 신뢰 제공
  2. 실행하는 1%가 사업 견인
  3. 전문가 = 가격 결정권
- 실행: 해외 연구 인용 / 학술 이론 / 전문 용어 적절히

### 5. 콜투액션 (CTA) ★ 핵심 룰
- 다음 액션 명시 (다음 영상·댓글·이메일·커뮤니티·독서·다운로드 등)
- **결제 페이지 직전까지는 상업적 티 X** (모든 게 무료처럼 느껴지게)
- **제공자 태도 (지배형) 유지** — 요청자 태도 (종속형) 절대 X

### 6. 마무리 후킹 (정점·종점 원칙)
- 카너먼: 마지막 감정이 가장 오래 기억
- 핵심: "내가 알려준 액션 취하면 손쉽게 다 가짐. 안 하면 큰 손해" 느낌
- **6상자 카피** 적용

## 3. 제공자 태도 vs 요청자 태도

모든 콘텐츠에서 **제공자 태도** 유지. 요청자 어휘 발견 시 즉시 교체.

### 어휘 매핑 표

| ❌ 요청자 (종속형) | ✅ 제공자 (지배형) |
|---|---|
| "꼭 읽어주시길 부탁드립니다" | "읽으셔야 불이익 없습니다" |
| "구매 방법은 아래와 같습니다" | "구매 방법은 아무나 찾을 수 없게 숨겨두었습니다" |
| "언제든 누구든 들어오실 수 있습니다" | "인원이 많아 조절 중입니다" |
| "포기 안 하시게 도와드리겠습니다" | "진입장벽은 점점 높아지니 자신 없으면 포기하세요" |
| "후회 없으실 거예요" | "망설여지면 사지 마세요. 다른 적임자에게 양보하세요." |
| "혹시 검토 부탁드려도 될까요?" | "검토하시고 가치 있다 판단되면 연락 주십시오" |
| "잘 부탁드립니다" | "이런 가치를 만들 수 있습니다" |
| "조금이라도 도움이 되었으면 합니다" | "이걸 모르면 N년은 손해입니다" |

### 패러다임 문장
- "어디, 네가 평범한 명청이들과 다른지 한번 볼까?"
- "딱히 필요하진 않은데, 원하면 도전해봐"
- (VacAX 톤으로 절제: 거만함 아닌 자신감)

## 4. 모호어 금지 룰

평가자(채용 담당자·고객·투자자)는 모호어를 **불가능한 것으로 간주**.

### 금지 어휘 → 대체

| ❌ 모호 | ✅ 명확 |
|---|---|
| "할 수도 있다" | "한다" / "안 한다" |
| "고려하고 있다" | "한다" / "5월까지 결정한다" |
| "검토 중" | "[날짜]까지 결정" |
| "가능하면" | 조건 명시 |
| "추후" | 구체 날짜 |
| "다양한" | 수치·예시 명시 |
| "많은" | "N개", "N%" |
| "효율적인" | 측정 가능 지표 |

### 계량화 원칙
계량화 가능한 모든 것은 수치로:
- ❌ "사용자가 많이 늘었다" → ✅ "월 사용자 3배 (300 → 900명, 60일)"
- ❌ "성과가 좋다" → ✅ "전환율 2.1% → 4.7%"

## 5. 수단 X 목표 O (출사표 이론)

사업·제품 설명 시 **기술·수단(X) 대신 고객이 얻을 결과·목표(O)** 앞세움.

| ❌ 수단 | ✅ 목표 |
|---|---|
| "해외 구매대행 강의" | "남의 물건으로 떼돈 버는 법" |
| "코딩/개발자 강의" | "2년 안에 연봉 7천 받는 법" |
| "재활 운동법" | "평생 20대 무릎 유지하는 법" |
| "Flutter 개발자" | "AX/AI 기획으로 [고객 성과] 만든 사람" |
| "AI 멀티 에이전트 스튜디오" | "퇴사 6개월 만에 AX/AI 기획자 합격하는 법" |
| "영어 발음 교정 앱 (VoiceFit)" | "녹음 한 번에 면접 영어 발음 평생 완성" |

## 6. N그래프 스토리텔링 (자기소개·이력서·면접)

본인 스토리는 **N그래프 3단계**로:

### N그래프 구조
1. **기대** — 처참한 현실에서 안간힘 써서 겨우 작은 성공
2. **좌절** — 어떤 계기로 큰 좌절·하락
3. **성공** — 자신만의 비결로 이전보다 훨씬 크게 성공

### 4종 비교
| 그래프 | 효과 |
|---|---|
| 하단 수평 (낮은 레벨 유지) | 희망 X |
| 상단 수평 (금수저) | 희망 O, 신뢰 X |
| 1차 직선 (꾸준 성장) | **진부함**. "운이 좋았겠지" |
| **N그래프** (좌절→성공) | **희망 + 깊은 신뢰** |

### 임용후 (CEO) N그래프 적용 예
- 기대: 공공 B2B 사업기획 2년 2개월 (안정적 직장)
- 좌절: 퇴사 + 실업급여 4개월 제약
- 성공: 1인 AI 멀티 에이전트 스튜디오로 4개월 만에 AX/AI 기획자 포트폴리오 완성

→ 면접·이력서·블로그·LinkedIn·자기소개서 모두 이 구조.
→ 1차 직선("꾸준히 성장했습니다") = 진부함 회피.
→ 좌절을 의도적으로 드러내야 신뢰.

## 7. VacAX 톤 안전장치 ⚠️

책의 원본 톤은 자극·악용 경계에 가까움. **VacAX는 신뢰·정직 톤 우선**. 다음 사용 자제.

### 사용 자제
- **패닉 이론** (Ch3.13) — 마지막 발자국 함정·강제적 죄책감. 단기 매출↑ 가능하나 신뢰 박살. 적용 X.
- **알라딘 이론** "당신은 적임자" — 진정성 동반 시만. 남발 X.
- **버스 이론** — 가짜 시간 제한·인원 제한 절대 X. 약속하면 진짜 지킬 것.
- **가짜 희소성**: "지금만 할인", "한정 인원" 같은 가짜 거짓말 X.

### 사용 시 검증 룰
- **후킹 전 가치 검증**: 카피가 6상자 다 통과해도, 실제 상품이 그 카피만큼 가치 있는지 자가 검증. 미달이면 카피 톤 낮춤.
- **악용 금지**: 책 자체에도 여러 챕터에 "악용 금지" 명시. VacAX 적용 시 그 정신 유지.
- **외부 시점 작성**: GitHub 공개 가능한 수준의 톤 (CLAUDE.md 운영 원칙 2번과 일치).

## 8. 적용 절차 (작성 → 자동 점검)

외부 노출 콘텐츠 작성 후 게시 전 자동 점검:

```
[ ] 1. 제목·헤드가 6상자 중 3개 이상 통과
[ ] 2. 전체 구조가 6단계 글공식 따름
[ ] 3. 제공자 태도 (요청자 어휘 0건)
[ ] 4. 모호어 없음 (할 수도 있다·고려 중·다양한 등 0건)
[ ] 5. 수단 X, 목표 O로 표현
[ ] 6. N그래프 적용 (자기소개·이력서·스토리 콘텐츠 한정)
[ ] 7. 안전장치 통과 (가짜 희소성·패닉 함정 0건)
```

체크 미통과 시 통과할 때까지 수정 후 게시.
