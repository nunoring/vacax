# 검증된 패턴 & 안티패턴

## 안티패턴 (하지 말 것)

**AP-001** 결정 변경 시 옛 코드 정리 안 하면 정합성 깨짐 → 결정 변경 시 "옛 결정 결과물 정리 체크리스트" 먼저

**AP-002** 새 보강 작업이 이전 보강 덮어쓰는 패턴 → 핸드오프에 "기존 N~M 결과 유지 의무" 명시 + 회귀 테스트 의무

**AP-003** 폴백에 필수 필드 누락 → 사용자 빈 화면 → 폴백은 production-ready 수준으로 전 필드 채울 것

**AP-004** 잠금 콘텐츠 블러만으로 가리기 → 전환 동기 약함 → "항목 리스트 + 1개 티저 노출" 패턴 사용

**AP-005** 이모지 3개 아이콘 시스템 = 자동생성 느낌 → 일러스트/사진/커스텀 아이콘 사용

**AP-006** AI 일반론 톤 → "레이어드컷은 윤곽 강조해요" 같은 표현 → 개인 관찰 의무 + 메커니즘 4요소 강제

**AP-009** 명세 매번 누적 확장 → 출시 못 함 → Phase 1(MVP) / Phase 2(v1.1) 분리 의무

**AP-010** AI 재시도 무제한 = 사용자 60초 대기 → 재시도 1~2회 제한, 폴백으로 즉시 처리

**AP-016** PowerShell ConvertTo-Json 한글 인코딩 깨짐 → DB 패치는 Node.js 스크립트 또는 Supabase SQL Editor 사용

**AP-017** 소형 LLM에게 복잡한 포맷 지시 → 무시함 → 포맷(이미지 배치, 마커 등)은 코드가 post-process로 처리

**AP-018** 외부 API 실패 시 "서버 탓"으로 단정 + 코드 면밀 검토 안 함 → 진짜 원인은 설계 미스 (timeout, maxTokens, schema 등) → 외부 탓 진단 전 코드 점검 의무. CEO가 "서버 아니다" 외쳤을 때 즉시 코드 재검토.

**AP-019** schema/maxTokens/timeout 튜닝 시 한 번에 하나씩 변경 + 빌드 + 검증 사이클 = 13차 빌드. 각 빌드 2~3분 + 사용자 인내심 소진. → **한 번에 변경 다 박고 stopwatch + stop_reason + token usage 모두 로깅 → 1차 빌드에서 전체 데이터 수집** 후 다음 결정.

**AP-020** Claude/LLM은 schema에 필드 있으면 maxTokens 한도까지 다 채워 응답. system prompt에 "X자 이내" 박아도 무시. → 응답 잘림 방지하려면 schema 자체를 슬림화 (필드 제거, 배열 길이 축소) 또는 maxTokens 늘리고 JSON balancer로 graceful 복구.

**AP-021** Pro 무거운 응답 1회 호출로 받음 → 매번 잘림/timeout. → 2단계 호출 (필수 콘텐츠 빠른 응답 + 부가 콘텐츠 백그라운드)로 분리. 첫 화면 진입 시간 절반으로 줄임.

**AP-022** 외부 서비스 API 상태를 학습 지식(스테일)으로 단정해 추천 → 티스토리 OAuth 살아있다 추천했으나 2024.2 종료 확인(웹 검색). → 외부 API/플랫폼 정책은 추천·설계 전 웹 검색으로 라이브 검증 필수. (한국 블로그 네이버·티스토리 공식 글쓰기 API 둘 다 없음 = 봇/수동만 가능)

## 검증된 패턴 (써먹을 것)

**VP-004** 프롬프트에 실제 예시 데이터 박아넣기 → GPT가 그 수준 흉내냄 (few-shot 강화)
- "메이크업 추천해줘" vs "형식: {제품명: 한율 부들밤 패드, 사용법: ...}" → 후자가 품질 3배

**VP-006** AI 응답 3중 안전망: 스키마 검증 → 재시도(1~2회) → 풍부한 폴백
- AI 기반 앱 필수 패턴

**VP-009** 오프라인 vs 앱 가격 비교 위젯 → Pro 가치 정당화
- "컨설팅 30만원 vs 앱 9,900원" 시각 비교

**VP-012** Phase 분할: 신규 통찰 → Phase 2로 → Phase 1은 출시 가능 최소 단위 유지

**VP-015** 이미지 슬롯 명시 방식 → LLM이 사진 내용 알고 글 씀
- `<사진1_음식>`, `<사진2_내부>` 형태로 프롬프트에 전달

**VP-016** 외부 API 등록 직후 즉시 키 검증 (헬스체크 API 호출 → 200 확인)

**VP-014** Claude Code = 코드 작업 + 즉석 멀티에이전트 회의 동시 가능
- "이 제품 [직군] 관점 보완점" 질문 하나로 코드베이스 아는 회의 소집
