# HairFit MVP Roadmap (상세판)

Updated: 2026-07-02

## Completion Definition (2단계)

**Level 1 — 상담 MVP 완성 (현재 목표):**
"미용사가 손님 앞에서 사진을 넣고, 3컷 또는 레퍼런스 결과를 보여주고, 얼굴 분석 근거로 짧게 상담한 뒤 기록까지 남길 수 있음." 완벽한 동일인 보존은 주장하지 않고 "상담용 얼굴 인상 유지"로 말한다.

**Level 2 — 상품화 (게이트 뒤):**
미용사 파일럿에서 재사용 의사·지불 의사가 확인된 경우에만 진입. 출시 결정이므로 `/council` 필수.

관련 패턴: ❌-05, ❌-06, ❌-09, ❌-10, ❌-14, ✅-01, ✅-10, ✅-12, ✅-16

## 현재 완성도 진단 (2026-07-02)

| 영역 | 진단 | 근거 |
|---|---|---|
| 기능 흐름 | ~85% | mock 전 흐름(홈→동의→사진→성향→모드→결과→저장→상세) 통과. 실제 업로드 QA만 미실행 |
| 생성 품질 | ~70% | synthetic 5장 통과(얼굴 유지 4/5 이상 4장). **실사 검증 0회 — 최대 리스크** |
| 디자인 | ~50% | 1차 리프레시 "너무 창백함" 판정. 2차 필요 |
| 상담 카피 | ~60% | 구조는 완성, 미용사 현장 말투 미반영 |
| 인프라/운영 | ~40% | 코드 폴더 git repo 아님, 배포 없음, localStorage only |

## 상품성 진단

- **포지션**: 소비자용 헤어 시뮬 앱(헤어핏·미러로이드류)은 다수. "미용사 상담 보조 + 얼굴분석 근거 + 상담 기록"의 B2B 상담툴 포지션은 상대적으로 빈 곳. 단, **수요 실증 0** — 미용사 파일럿 전까지 상품성은 가설.
- **단가 구조**: 실측 $0.04/컷, 상담당 3컷 = ~$0.12(약 165원). 하루 10상담 = $1.2. 월 구독 3~5만원 모델이면 마진 성립 — 비용은 상품성 걸림돌 아님.
- **핵심 리스크 2개**: ① 실사(현장 조명/각도) 품질 미검증, ② 미용사가 실제 상담 중 꺼내 쓰는지 미검증. 이 둘이 상품성의 전부. 디자인·기능 추가는 이 둘 앞에서는 부차적.

## Phase 1. 디자인 2차 패스 (2~3일)

목표: 초딩 느낌과 창백함 동시 제거. "흰 병원 접수표"가 아니라 "살롱 상담툴".

해야 할 일:
- 디자인 토큰 재정의: 배경(딥 뉴트럴 1색), 표면 2단, 텍스트 3단, 액센트 1색, 상태색. 다크/라이트 중 **하나로 고정**, 혼합 금지.
- 홈: 최근 기록의 결과 사진이 첫 시선. CTA 1개만 강조.
- 결과: 이미지 카드 최대폭, "손님에게 이렇게" 타이포 위계 최상단, 근거는 표/접힘 유지.
- 동의/사진/성향/분석중/저장/기록/상세/근거 모달 동일 토큰 적용.
- 금지 유지: 카드 과다, 칩 과다, 큰 라운드, 무의미 장식, 이모지 장식(❌-05).

Pass 기준:
- CEO가 "초딩 느낌도 아니고 창백하지도 않다"고 판단.
- 홈과 결과 화면만 봐도 미용사용 상담툴 정체성이 보임.
- `npm.cmd run build` 통과.

## Phase 2. 실제 업로드 QA (0.5일, 수동)

목표: 샘플 사진이 아니라 실제 파일 업로드가 상담 시작부터 저장까지 문제 없는지 확인.

해야 할 일:
- 실제 화면에서 사진 파일 1장 수동 업로드 (자동화 표면에서 파일 주입 미지원 — 수동 필수).
- 업로드 사진 기준 얼굴 분석 → 결과 카드 → 저장 상세 확인.
- 원본 사진 저장 OFF 기본값, 삭제 요청/영구 삭제 흐름 확인.
- **모바일 실기기 1회 추가**: file input `capture` 카메라 경로는 데스크탑과 동작이 다름. 미용사 실사용 환경 = 폰.

Pass 기준:
- 업로드 포함 흐름: 사진 → 성향 → 모드 → 결과 → 저장 → 상세 통과 (데스크탑 + 모바일 각 1회).
- 텍스트/이미지 겹침 없음, console error/warn 없음.

## Phase 3. 실사 품질 검증 (1~2일, 예산 $1 미만) — 신규, 최우선 리스크

목표: synthetic 통과 ≠ 실사 통과 (LEARNINGS 확인 사항). 상품성 판단의 근거를 만든다.

해야 할 일:
- 동의 확보된 실제 얼굴(본인/지인, 동의 기록 남김) 2~3장 × 스타일 3종(짧은/중간/긴 기장 각 1) = 6~9컷 실제 생성.
- 현장 조건 1장 포함: 조명 나쁨/각도 틀어짐/배경 복잡.
- 랜덤 인터넷 얼굴 금지 (기존 테스트 사진 정책 유지).
- 1컷 단위 실행, 호출 후 즉시 `hairfit:mock` 재잠금 (기존 SOP).
- 실패 시: `photoQuality` 가드 강화(재촬영 안내 문구) + 프롬프트 보강 후 1회만 재시도. 폴리시 루프 금지.

Pass 기준:
- 얼굴 인상 유지 4/5 이상이 인물 2/3 이상.
- 상담 사용성 3/5 이상 전부.

KILL 분기:
- 재시도 포함 2회 미달 → "실제 합성" 주장 강등, "스타일 미리보기 + 얼굴분석 상담 근거" 중심으로 포지션 피봇. 로드맵 Phase 5 이후 그대로 진행 (앱 완성 자체는 유효).

## Phase 4. 미용사 카피 반영 (1일, CEO 대기)

목표: 상담 문장을 현장 말투로.

해야 할 일:
- "손님에게 이렇게" 문장 길이/말투 조정.
- 얼굴 분석 근거를 미용사가 실제로 말할 수 있는 1~3줄로 압축.
- 3컷마다 같은 말 반복 제거, 카드별 상담 포인트 차별화.
- 시술 전 확인 포인트를 미용사용 언어로.

Pass 기준:
- 미용사가 읽고 "손님 앞에서 말할 수 있다"고 판단.
- 결과 카드 3개가 서로 다른 상담 포인트.
- 일반론/AI 티 문장 제거.

블로킹 아님: 피드백 지연 시 Phase 5~7 먼저 진행, 도착 즉시 반영.

## Phase 5. 실제 생성 가드 확정 (0.5일)

목표: 실제 생성은 열어두되 비용/실패/과장 리스크 차단.

해야 할 일:
- 카드별 실제 생성 confirm 문구 재확인.
- 실패 시 fallback 문구/화면 상태 확인.
- `gpt-image-2` aspect `auto` 유지, 긴 기장/리프 계열 hair-crop reference 유지.
- 세션당 실제 생성 상한 카운터 추가 (예: 10컷/세션, 초과 시 차단 문구) — 파일럿 중 폭주 방지.

Pass 기준:
- 실제 생성 OFF 시 비용 0. 켜도 1컷 단위만.
- 호출 후 `hairfit:mock` 재잠금 확인.
- 세션 상한 동작 확인.

## Phase 6. 안정화 스윕 (1일)

목표: 데모/파일럿 중 깨지는 곳 제거.

해야 할 일:
- 모바일 폭/데스크탑 phone-frame 양쪽 확인.
- 새 상담, 뒤로가기, 조건 수정, 다시 생성, 저장, 상세, 삭제 흐름 점검.
- **localStorage 용량 점검**: 결과 이미지 base64 저장 시 5MB 한도 — 기록 50건+ 시나리오에서 저장 실패/깨짐 확인, 필요 시 오래된 기록 정리 안내 또는 이미지 압축.
- 빈 상태/실패 상태/mock 상태 문구 정리.

Pass 기준:
- `npm.cmd run typecheck` / `build` / `hairfit:check-assets` 통과.
- 핵심 mock flow 통과, 기록 50건에서도 홈/기록 화면 정상.
- proxy `realEnabled=false`로 마감.

## Phase 7. 데모 배포 (0.5~1일)

목표: 미용사가 자기 폰에서 열어볼 수 있는 URL. 파일럿 전제조건이자 포트폴리오 자산.

해야 할 일:
- mock 모드 정적 배포 (Vercel/Netlify). proxy 없이 mock only — 배포본에서 실제 생성 불가 = 비용 0 보장.
- 키/환경변수 미포함 검증.
- 모바일 브라우저(iOS Safari/Android Chrome) 1회씩 확인.

Pass 기준:
- 외부 URL에서 mock 전 흐름 동작 (업로드 포함).
- 배포본에서 유료 호출 경로 없음.

## Level 1 완성 판정 (7개 모두 true)

1. CEO가 홈/결과 디자인을 "손님 앞에서 켜도 덜 창피함" 이상으로 인정.
2. 사진 업로드 포함 상담 흐름 통과 (데스크탑 + 모바일).
3. AI 추천 3컷과 레퍼런스 선택 흐름 모두 저장까지 통과.
4. **실사 기준** 생성 품질이 "상담용 얼굴 인상 유지"로 설명 가능 (또는 KILL 분기 후 미리보기 포지션 확정).
5. 비용 가드 + mock 재잠금 + 세션 상한 유지.
6. 미용사 피드백 기반 상담 문장 1차 반영.
7. 외부 URL 데모 존재.

예상 소요: Phase 0~7 실작업 6.5~9일 (하루 5~6h 기준 약 1.5~2주). Phase 4는 피드백 도착 시점 의존.

## Phase 8. 미용사 파일럿 (달력 1~2주, 상품성 검증)

목표: 상품성 가설("미용사가 상담 중 실제로 쓴다")을 데이터로.

해야 할 일:
- 미용사 1~3명에게 데모 URL 전달, 실제 상담 5회 이상 사용 요청.
- 측정 항목 고정: ① 상담 중 실제로 꺼냈는가(회수), ② 손님 반응, ③ 재사용 의사(y/n), ④ "월 3만원이면 쓰겠나"(y/n).
- 피드백을 `EVALS.md`에 기록.

GO/KILL 기준 (사전 고정):
- **GO**: 재사용 의사 2/3 이상 + 지불 의사 1명 이상 → Phase 9 진입.
- **KILL**: 재사용 의사 0 → 상품화 중단, 포트폴리오 자산으로 종결 (케이스 스터디 작성으로 전환).
