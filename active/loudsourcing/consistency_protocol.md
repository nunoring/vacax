## 0. 툴 벤치마크 (먼저 — 툴은 가정 말고 실측으로 확정)

목적: 이미지·영상 주력 툴을 동일 조건 슛아웃으로 결정. nano banana 주력 지위 철회 후 재선정.

**후보**
- 이미지: MJ v7(cref/sref) / Leonardo / nano banana(재검증)
  - ❌ 제외: 로컬 Stable Diffusion + DreamBooth/LoRA — 셋업·학습 = 시간트랩. (GPU+시간 여유 + 그 스킬 자체가 포폴 자산일 때만 재고)
- 영상: Runway / Kling / Veo2

**방법 (동일 조건 슛아웃)**
1. 레퍼런스 1인 5포즈 고정 (고해상도)
2. 동일 스타일키트(§1) + 동일 프롬프트템플릿(§2) + seed 고정 + 해상도 ≥1K
3. 각 툴 3컷씩 생성

**평가표 (모드어스급 바, benchmarks.md 기준)**

| 항목 | 통과 기준 |
|---|---|
| 얼굴 아티팩트 | 눈·손·치아·주얼리 결함 0 |
| 피부 실사감 | 모공·결 살아있음, 플라스틱/왁스 X |
| 컷 간 인물 유사도 | 동일인으로 인식 (육안 + 차이맵) |
| 업스케일 후 품질 | 4배 업스케일 시 붕괴 없음 |
| 프롬프트 순응도 | 지시한 무드·구도 반영 |

→ 결과 비교표 → 주력 툴 확정 → STATE.md 「생성 스택」 갱신.

## 1. 스타일 키트 (.STYLE) — 콘테스트마다 1장, 고정

| 항목 | 값 |
|---|---|
| 팔레트 | HEX 3개 (dominant / accent / base) |
| 폰트 | (구글 무료, 이미지 텍스트는 후처리용) |
| 비율 | 이미지 / 썸네일 / 영상 각각 |
| 톤 키워드 | 3개 (예: 따뜻함·디테일·럭셔리) |
| 조명 | (예: soft rim light) |
| seed 규칙 | 핵심 컷 seed 고정값 |
| 캐릭터 토큰 | <project_avatar> (인물 락용) |
| 모델·버전 | 확정 툴 + 버전 명시 |

이 블록을 **모든 생성 프롬프트 상단에 상수로 박는다.** 단어 하나 바꾸면 일관성 붕괴 → 변경 금지.

## 2. 프롬프트 템플릿 (.PROMPT)

```
[STYLE_BLOCK] + [피사체 묘사] + [카메라 앵글/무브] + [조명] + [무드] + [색감] + [품질태그]
+ seed:{고정값} + model:{확정툴+버전}
```

**툴별 일관성 토큰** (슛아웃 후 확정된 툴만):
- MJ: `--cref [ref] --sref [style] --sw [weight]`
- Leonardo: custom embedding 토큰 + consistency 설정
- nano banana: reference image + 동일 seed

**⚠️ 이미지 속 텍스트는 AI 생성 금지 → Figma(MCP) 후처리 삽입.** AI 글자 깨짐 = 즉시 감점.

- 레퍼런스 이미지 = `ref_v01.png` 버전태그. **절대 덮어쓰기 금지.**

## 4. 자산 레지스트리 (재현성 = 콘테스트 통과조건)

매 생성물 1줄 기록:

| asset_id | 콘테스트 | 프롬프트 | seed | model/ver | 툴설정 | ref파일 | 결과 | QA |
|---|---|---|---|---|---|---|---|---|

- **재현 테스트**: 같은 메타로 재생성 → 80%+ 유사 시 통과.
- ⚠️ 라우드/공공 콘테스트 다수가 **AI 작업기록(도구·프롬프트·후처리) 필수 제출** → 이 레지스트리가 곧 제출 서류 + 포트폴리오 자산(workflow_v1 §F).
