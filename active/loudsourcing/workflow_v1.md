## 1. 전략적 포지셔닝

### 1.1 핵심 명제 (출사표 이론 적용)

대부분의 AI 크리에이터는 "AI 툴을 쓸 수 있는 사람"으로 스스로를 정의한다. 이는 **수단 포지셔닝**으로, 툴 숙련도 경쟁에 자신을 가둔다.

VacAX의 포지셔닝:

> **"브랜드가 전달하고자 하는 가치를 AI 워크플로우로 시각화하는 기획자"**

이 정의는 세 가지 경쟁 우위를 동시에 확보한다:
- 심사 기준 창의성(30점) 영역에서 기술자와 완전히 다른 차원의 접근
- "AI를 쓸 줄 안다"가 아닌 "AI로 무엇을 만들어야 하는지 안다"는 포지셔닝
- AX/AI 기획 이직 포트폴리오와 동일한 방향성 → 공모전 참여 자체가 포트폴리오 자산

### 1.2 경쟁 구조 분석

라우드소싱 AI 공모전 심사 배점 기준 (공개 자료 기반):

| 심사 항목 | 배점 | 기술자의 현실 | 기획자의 현실 | **VacAX** |
|---|---|---|---|---|
| 창의성 | 30점 | 브랜드 의도 해석 미흡, 10~15점 | 기획력 강점, 25~30점 | **25~30점** (CEO 기획력) |
| AI 활용도 | 30점 | 25~30점 | 툴 미숙, 10~15점 | **25~30점** (Claude 디렉션 + 작업기록) |
| 완성도 | 30점 | 25~30점 | 생성 품질 미흡, 10~15점 | **20~25점** (툴 숙련도 향상 중) |
| 기타(브랜드 정합성) | 10점 | 5점 내외 | 8~10점 | **8~10점** |
| **예상 합계** | **100점** | **~70점** | **~50점** | **~85점** |

**VacAX의 구조적 우위는 완성도를 제외한 3개 항목에서 발생한다.** 완성도는 툴 숙련도에 비례하므로, 반복 출품을 통한 학습 곡선 단축이 유일한 해결책이다.

### 1.3 운영 구조 (분업)

| 담당 | 역할 | 근거 |
|---|---|---|

## 2. 공모전 스크리닝 기준 (GO / KILL)

신규 공모전 발견 시 아래 기준으로 72시간 이내 GO / KILL 판정.

### 2.1 즉시 GO 조건 (4개 중 3개 이상 충족)

| 항목 | 기준 | 이유 |
|---|---|---|
| 참여작 수 | ≤ 200개 | 공정한 경쟁 가능 구간. 초과 시 당선 기댓값 급락 |
| 마감까지 잔여일 | ≥ 5일 | 브리프 분석 2일 + 생성/편집 2일 + 버퍼 1일 |
| AI 허용 명시 | 명문화 여부 | 미명시 시 제출 후 탈락 리스크 |
| 브리프 주제 | 기획력으로 풀 수 있는 주제 | 공공 가치·브랜드 스토리·사회적 메시지 등. 단순 기술 시연은 차별점 발휘 불가 |

### 2.2 즉시 KILL 조건 (1개라도 해당 시)

- 참여작 500개 초과 (이미 레드오션화)
- 마감 3일 이하 (품질 보장 불가)
- AI 사용 금지 또는 모호한 공모전
- 순수 디자인/일러스트/로고 카테고리 (AI 차별점 무의미)
- 출품 후 저작권이 100% 주최사에 귀속되는 조건 (포트폴리오 활용 불가)

### 2.3 우선순위 공모전 유형

1. **공공기관 AI 홍보영상**: 심사가 "브랜드 기획 이해도" 대신 "주제 전달력 + AI 활용 명시도"로 구성 → CEO 기획력 직접 연결
2. **브랜드 숏폼 광고**: 브랜드 레퍼런스 분석 선행 시 창의성 30점 독점 가능
3. **참여작 100개 이하 공모전**: 당선 기댓값이 구조적으로 높음

## 3. 공모전 실행 프로세스

### PHASE A — 브리프 분석 및 전략 설계 (Claude 주도, 1~2일)

#### A-1. 브리프 해독

브리프에서 추출해야 할 항목:

| 항목 | 추출 내용 | 목적 |
|---|---|---|
| 명시적 요구 | 주제, 포맷, 러닝타임, 해상도, 제출 방식 | 실격 방지 |
| 암묵적 요구 | 브랜드 무드, 기존 광고 레퍼런스, 타겟 고객층 | 창의성 점수 결정 |
| 심사위원 관점 | "이 브랜드가 이 공모전으로 얻고 싶은 것은 무엇인가" | 기획자와 기술자의 차이 |
| 금지 항목 | 특정 표현, 타사 제품, 정치적 내용 등 | 실격 방지 |

#### A-2. 경쟁 차별화 각도 설정 (출사표 이론 적용)

일반적 접근 (기술자 패턴): AI 생성 영상의 비주얼 완성도에 집중.  
VacAX 접근: 브랜드가 전달하려는 **핵심 가치 하나**를 찾고, 그것을 의외의 방식으로 시각화.

예시 — 식품 브랜드 "원료에서 식탁까지" 주제:
- 기술자 접근: 예쁜 식재료 AI 영상
- VacAX 접근: "원료를 선택하는 사람의 철학"을 영상에 담는 내러티브 구조

#### A-3. 스토리라인 설계 (뇌과학 글공식 6단계 적용)

AI 영상 공모전 스토리라인은 6단계 구조로 역설계한다. 심사위원은 수백 개의 작품을 보는 인간이다. 첫 3초에 후킹되지 않으면 나머지는 없다.

| 단계 | 영상 내 위치 | 설계 원칙 |
|---|---|---|
| **1. 메인 후킹** | 0~3초 | 6상자 카피라이팅 기반 첫 장면/카피. "어, 이게 뭐지?" 반응 유발 |
| **2. 서브 후킹** | 4~10초 | 영상 끝까지 봐야 하는 이유. 감정 엔커링 (음악·색감·분위기 고정) |
| **3. 효과 입증** | 11~20초 | 브랜드가 주장하는 가치를 시각적으로 직관 증명. 설명 X, 보여주기 O |
| **4. 전문가 포지셔닝** | 21~40초 | 브랜드의 깊이와 철학. 시청자는 내용보다 "역시 대단한 브랜드다" 감정 수용 |
| **5. CTA** | 41~50초 | 시청자가 해야 할 다음 행동 암시 (브랜드 탐색, 구매, 공유) |
| **6. 마무리 후킹** | 51초~끝 | 마지막 감정이 가장 오래 기억됨. 처음보다 강한 인상으로 마감 |

30초 영상의 경우 각 단계를 5초 내외로 압축. 6단계 구조는 유지.

### PHASE B — 프롬프트 설계 (Claude 주도, 반일)

#### B-1. 장면 분해

러닝타임을 6단계에 맞게 장면으로 분해한 후, 장면별 프롬프트를 개별 작성.

장면별 프롬프트 구조:
```
[피사체 묘사] + [카메라 앵글/무브먼트] + [조명] + [분위기/무드] + [색감] + [품질 태그]
```

품질 태그 기본 세트 (Kling AI 기준):
```
cinematic quality, 4K, smooth motion, no artifacts, professional color grading,
commercial advertisement style, [브랜드 키워드]
```

#### B-2. 작품 제목 및 설명 작성 (6상자 카피라이팅 적용)

제출 시 작품 제목과 설명은 심사위원의 첫 인상을 결정한다.  
6상자 기준 최소 3개 이상 통과를 목표로 작성.

| 상자 | 적용 방법 |
|---|---|
| 욕구성 | 브랜드가 고객에게 주는 핵심 가치와 연결 |
| 날먹성 | "단 [시간]에 [결과]를 담았다" 등의 임팩트 |
| 충격성 | 의외의 시각화 각도 명시 |
| 희소성 | 이 작품만이 가진 접근법 |
| 특정성 | 수치, 구체적 기술, 명확한 개념 |
| 정체불명 | 독자적 표현·명칭 사용 (모방 불가한 인상) |

### PHASE C — 생성 실행 (CEO 주도, 1일)

#### C-1. 도구 스택

| 용도 | 도구 | 플랜 | 월 비용 |
|---|---|---|---|
| AI 영상 (주력) | Kling AI | Standard | $6.99 ≈ 1만원 |
| AI 영상 (보조·무료) | Veo 2 (Google AI Studio) | 무료 (하루 10개 한도) | 0원 |
| AI 이미지 | Leonardo.ai | Free (150토큰/일) | 0원 |
| 편집 | CapCut | 무료 | 0원 |
| **월 합계** | | | **≈ 1만원** |

예산 상한 5만원 대비 잔여 4만원은 공모전 품질 임계점 도달 후 추가 플랜 업그레이드에 유보.

#### C-2. 생성 절차

1. 장면별 프롬프트를 Kling AI에 순차 입력
2. 장면당 최소 3개 변형 생성 (seed 번호 기록 필수)
3. Veo 2로 동일 장면 병행 생성 → 비교 후 품질 우수한 것 선택
4. 생성 결과 전체 스크린샷 및 원본 파일 보관 (작업기록용)

| 항목 | 기준 | 결과 |
|---|---|---|

### PHASE F — 작업기록 작성 (Claude 주도, 반일)

#### F-1. 작업기록의 이중 목적

작업기록은 공모전 제출 서류이자 **포트폴리오 메아리 자산**이다.  
심사위원에게는 AI 활용도(30점)의 근거가 되고, 외부에 공개 시 "이 사람은 AI 워크플로우를 설계할 수 있는 사람"이라는 신뢰 자산이 된다.

#### F-2. 작업기록 구성

```
[제목] [공모전명] AI 영상 제작 워크플로우

1. 전략 설계
   - 브리프 해독 결과 (암묵적 요구 포함)
   - 차별화 각도 선정 근거
   - 스토리라인 6단계 설계

2. 생성 과정
   - 사용 도구: Kling AI Standard / Veo 2 (Google AI Studio)
   - 장면별 프롬프트 전문 (입력값 그대로)
   - 생성 시도 횟수 및 선택 근거

3. 후처리
   - 편집 도구: CapCut
   - 적용 효과 및 선택 이유
   - 최종 포맷 스펙

4. 품질 검증
   - QA 체크리스트 결과
   - 재생성 발생 장면 및 수정 내용
```

#### F-3. 포트폴리오 자산화 기준

작업기록 완성도가 포트폴리오 활용 가능 수준인지 판단 기준:

- 타인이 동일 결과를 재현할 수 있을 정도의 상세도
- 단순 절차 나열이 아닌 **선택 이유**가 포함됨
- 스텝 5개 이상

## 4. 포트폴리오 자산화 전략 (연쇄 레버리지)

공모전 참여 자체가 무료 시장 검증이자 포트폴리오 축적 수단이다.

```
콘테스트 출품
    → 입상 시: 라우드소싱 프로필 노출 + 신뢰 자산 (메아리)
    → 미입상 시: 작업기록 포트폴리오로 보존 (AI 워크플로우 설계 능력 증명)
    → 양쪽 모두: AX/AI 기획 이직 포트폴리오에 "실전 파이프라인 구축" 항목 추가
```

**첫 입상이 두 번째 의뢰를 만들고, 두 번째 의뢰가 세 번째 입상을 만든다.**  
콘텐츠 품질보다 워크플로우 문서화 수준이 이 사이클의 속도를 결정한다.
