# VoiceFit 완성 로드맵 (2026-07-02)

## "완성"의 정의

기능 100%가 아님. 아래 4개 충족 = 완성:
1. 말투 90점급(또는 데이터로 보정된 기준) + 사진 포함 글이 안정 생성
2. 티스토리(쿠팡 큐레이션) / 네이버(비상업 방문후기) 주기 발행 운영
3. 발행→노출→클릭→수익 측정 데이터 축적
4. 포트폴리오 케이스 스터디로 자산화 (AX/AI 기획 이직용)

근거: D-009(프라이빗), D-013(반자동 발행). SaaS 제품화는 폐기된 C안 — 이 앱의 상품성은
"판매용 SaaS"가 아니라 (a) 본인 부업 수익 도구 + (b) 포트폴리오 자산. 로드맵도 여기에 정렬.

## 상품성 판정 (현재)

| 축 | 판정 | 근거 |
|---|---|---|
| SaaS 판매 | 낮음 (의도적 보류) | 발행 API 부재(티스토리 API 종료, 네이버 미제공) → 풀자동 불가 천장. 네이버 D.I.A+ 제재 리스크 |
| 개인 부업 수익 | 중간 (미검증) | 쿠팡 파트너스 수익 = 트래픽/알고리즘 의존 → 본수익원 부적격, 실험으로는 유효. 실발행 0건이라 데이터 없음 |
| 포트폴리오 | 높음 | 말투 지문 분석 + 점수 게이트 + 2-pass 변환 + 채널 규제 대응(D-011/012)은 차별화된 AX 스토리 |

## 완성도 판정 (현재 92%의 실체)

- 92% = "로컬 개발 루프" 완성도. 제품 운영 완성도는 ~60%.
- 됨: 크롤링→말투 지문→9섹션 생성→채점→재생성→감사→발행 게이트 전 루프 (로컬 fallback)
- 안 됨: 실발행 0건, 실사진 Vision 검증 미완, Supabase 다운, 쿠팡 API 키 없음, 90점 미도달(82~84), 7/1 코드 전량 미커밋, 키 노출 이력 미회전
- Vercel 배포본은 5월 버전. 현 로컬 파일 fallback 구조는 serverless에서 작동 불가 → Supabase 복구 전 재배포 무의미

## Phase 3 — 운영 데이터 축적 + GO/KILL 판정 (2~4주, 타 작업과 병행)

- 발행 리듬: 주 3~5건 (채널 혼합)
- Supabase 복구 시 로컬 데이터 마이그레이션 + DB 흐름 재검증 → Vercel 재배포 의미 생김
- **판정 게이트 (4주차)**:
  - KILL: 네이버 노출 0 또는 제재 신호 + 티스토리 클릭 0 → 채널 전략 폐기, 포트폴리오 정리로 직행
  - PIVOT: 한 채널만 생존 → 생존 채널 집중
  - CONTINUE: 클릭/수익 발생 → 발행량 증대
- 판정 무엇이든 포트폴리오 가치는 유지됨 (실험 자체가 스토리)

## Phase 4 — 쇼츠·멀티채널 확장 (조건부)

**조건: Phase 3 CONTINUE 판정일 때만.** 아니면 영구 잠금 유지.

- 릴스 캡션 운영 개시

## Phase 5 — 포트폴리오 자산화 (전 구간 병행, 최종 마감 ~3일)

- 날것 과정 기록 유지: 도전→깨진 지점→교훈 (LEARNINGS.md 이미 양호 — 미화 금지)
- 케이스 스터디 문서: 말투 지문 분석 / 75점 게이트 / 2-pass 변환 / few-shot 장르 매칭 / 채널 규제 대응 — "AI티 문제를 시스템으로 푼" AX 스토리
- 수치 포함: 42→78→84점 개선 곡선, 테스트 48개, 발행/클릭 데이터
- 데모: 스크린샷 + 짧은 영상 (프라이빗 배포 유지)

## 타임라인 요약 (하루 5~6h)

```
Week 1     : Phase 0 (1~2일) + Phase 1
Week 2     : Phase 2
Week 3~6   : Phase 3 (병행 — 발행은 하루 1h 내, 나머지 시간 타 프로젝트)
Week 6     : GO/KILL 판정 → Phase 4 or 포트폴리오 직행
상시       : Phase 5 기록
```
