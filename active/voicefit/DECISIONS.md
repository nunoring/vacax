## D-020: 쇼츠 영상 MVP는 로컬 FFmpeg로 시작

- 날짜: 2026-07-01
- 상태: 확정
- 선택: Remotion/CapCut 연동 전, `쇼츠 제작 지시서`를 render plan JSON으로 바꾸고 로컬 FFmpeg로 1080x1920 MP4 샘플 생성
- 이유: 현재 목표는 완성 편집기가 아니라 자동 쇼츠 제작 흐름의 작동 증명. FFmpeg가 로컬에 설치돼 있고 새 런타임 의존성 없이 검증 가능
- 제한: 현재 결과물은 무음 텍스트 카드. 실제 발행 품질에는 상품 이미지/B-roll/TTS 보강 필요

## D-021: VoiceFit 본체는 말투 지문 분석

- 날짜: 2026-07-01
- 상태: 확정
- 선택: 말투 분석은 톤 요약이 아니라 이모지 타이밍, 사진 타이밍, 띄어쓰기, 줄넘김, 문장부호, AI티 위험 요소까지 잡는 지문 분석으로 운영
- 이유: 블로그 자동화의 성패는 "생성됨"이 아니라 "AI가 쓴 티가 안 남"에 있음. 균일한 문단·사진마다 같은 코멘트·이모지 남발은 바로 AI티로 보임
- 제한: 맞춤법 교정과 후킹을 하더라도 원문 블로거의 호흡을 지워버리면 실패

## D-022: 말투 점수 75점 미만은 발행 실패로 취급

- 날짜: 2026-07-01
- 상태: 확정
- 선택: 말투 점수 75점 미만이면 생성 화면 검수 이동과 검수 화면/API 발행을 막고 재생성 또는 수정 후 재채점을 요구
- 이유: VoiceFit의 핵심 가치는 자동 발행이 아니라 AI티를 줄인 블로그 자동화. 낮은 점수 글을 그대로 발행하면 말투 분석 기능의 신뢰가 깨짐
- 제한: 75점은 MVP 기준. 실제 운영 데이터가 쌓이면 점수 기준과 fail 항목을 조정
