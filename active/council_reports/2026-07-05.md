## 결정 1 — HairFit: Codex 전담 헤드 전환

**결정 내용**
Codex가 HairFit 전담 헤드 역할. Claude Code는 보조. HairFit = "미용사 상담 보조 + 얼굴분석 근거" 차별화.

### 🔵 strategist
- "미용사 상담 보조 + 얼굴분석 근거" 포지셔닝은 시장 차별화 가능
- **문제: Codex 선택의 시장 근거 없음.** HairFit 핵심은 AI 미용 분석 품질인데, Codex가 Claude보다 나은 이유를 데이터로 뒷받침 못 하면 실행 불가
- "전담 헤드"가 제품 기획 AI인지, 코드 생성 AI인지, 분석 AI인지 모호 → 역할 정의가 선행돼야 전략 판단 가능

### 🟡 validator
- 통과 기준 없음. "Codex가 낫다"는 비교 실험 수치 미명시
- Codex vs Claude 성능 비교 기준(미용 상담 품질, 코드 생성 정확도 등) 수치화 필요
- 기준 없는 AI 전환 = ❌-XX 패턴 재발 위험

### 🟢 builder
- "Codex"가 OpenAI Codex인지 다른 툴인지 불분명 — 구현 스펙 확정 불가
- 헤드/보조 역할 분리 시 최종 판단 권한 충돌 발생 가능 (누가 최종 코드 리뷰?)
- "미용사 상담 보조 + 얼굴분석 근거" 차별화가 AI provider 선택의 직접 근거가 되는 경로 불분명

### 🔴 growth
- AI provider 전환이 마케팅 차별화에 기여하는 경로 미설명
- 포트폴리오 스토리 일관성 리스크: HairFit은 신규 프로젝트인데 AI 전환 반복이 이미 시작되면 "결정력 부족" 인상

**최종 판정: ⚠️ 재검토 권고**
- Codex가 어떤 툴인지 정의 + HairFit에서 Claude 대비 무엇이 나은지 측정 기준 1개 이상 명시 후 재확인 필요
- 차별화 방향("미용사 상담 보조")은 GO — AI provider 선택만 재검토

## 결정 2 — VoiceFit D-015: 커머스 패키지 멀티채널 묶음

**결정 내용**
블로그 본문 하나 → 블로그/쇼츠/릴스/CTA/고지/실험 가설/체크리스트 9섹션 패키지. "가치는 생성량보다 채널별 검수·복사·발행 흐름에 있음."

### 🔵 strategist
- 방향 자체 맞음. 멀티채널이 1인 콘텐츠 작업대의 차별점
- **리스크: MVP 미완성 상태에서 9섹션 확장 = 완주 지연 위험.** "흐름이 가치"라면 흐름 자체가 먼저 검증돼야 함
- 1개 채널 완주 → 검증 → 확장 순이 전략적으로 맞음

### 🟡 validator
- 성공 기준 없음. "9섹션 패키지가 발행 흐름을 실제로 빠르게 하는가" 미검증
- 측정 기준 제안: 1개 블로그 글 생성~발행 시간 단축률, 패키지 중 실제 사용되는 섹션 수

### 🟢 builder
- 9섹션 묶음 = UX 과부하 위험. 사용자(CEO)가 한 화면에서 9개를 검토하는 흐름은 인지 부하 높음
- 쇼츠 지시서 → FFmpeg → MP4까지 파이프라인을 한 번에 묶으면 디버깅 복잡도↑
- **권장: 섹션 추가보다 1개 채널 완주 UX 먼저 완성 → 점진 확장**

### 🔴 growth
- 멀티채널 커버리지 = 경쟁 차별점으로 활용 가능
- 단 채널별 성과 추적 없으면 "더 많이 만든다"가 실질 수익으로 연결 안 됨
- CTA/고지 섹션 포함은 어필리에이트 전환율 추적에 필수 — 방향 맞음

**최종 판정: ✅ GO 유지 (조건부)**
- 방향은 맞음. 단 9섹션 전체 완성 전, 블로그 1개 채널 완주 UX 먼저 검증 필요
- 성공 기준(완주 시간, 사용 섹션 수) 1회 측정 후 다음 채널 확장 권고

## 결정 3 — VoiceFit D-021: VoiceFit 본체 = 말투 지문 분석 재정의

### 🔵 strategist
- "AI가 쓴 티가 안 남" = 네이버/티스토리 AI 제재 환경에서 시장 필요성 명확
- 경쟁사(ChatGPT 직접 사용자)와의 차별점 선명 — GO 신호
- **리스크: "이모지 타이밍, 사진 타이밍"이 기술적으로 구현 가능한지 spec 없음**

### 🟡 validator
- D-022 연결로 75점 기준 있음 → 성공 기준 충족 ✅
- "이모지 타이밍, 사진 타이밍" 측정 방법이 아직 spec 미완성이면 구현 전 정의 필요
- 현재 데이터: 72점 고착 → D-023으로 원인 분석됨. 분석 → 수정 → 재측정 루프 갖춰짐 ✅

### 🟢 builder
- 6개 이상 지표를 동시에 잡으면 복잡도 폭발
- **권장: 1차 MVP는 가장 영향력 큰 지표 2~3개로 좁히고 점진 추가**
- "원문 블로거의 호흡을 지워버리면 실패" 제한 → 검수 UX에서도 명시적으로 노출 필요

### 🔴 growth
- "AI탐지 75점 이상만 발행" = 마케팅 수치로 활용 가능
- "이모지 타이밍까지 잡는다" = 블로거 고급 니즈 겨냥 — 틈새 마케팅 각도 있음

**최종 판정: ✅ GO 유지**
- 방향 명확, 검증 체계(75점 기준) 갖춤
- 구현 시 지표 우선순위 명시 필요

## 결정 4 — VoiceFit D-023: 상품글 처리 흐름 전환

**결정 내용**
상품글 처리 = `정보 초안 → 광고문 제거/완화 → 2-pass 말투 변환 → 채점` 분리.
원인: `CURATION_COPY`/`HOOK_RULES`가 광고 문법을 우선해 말투 지문보다 광고 구조가 앞섬 → 72점 고착.

### 🔵 strategist
- 72점 고착이라는 실데이터 있음 + 원인 분석 명확 = 데이터 기반 결정 ✅
- "채점기는 먼저 고치지 않는다" = 검증 순서가 맞음. 섣부른 채점 기준 변경 방지

### 🟡 validator
- 통과 기준: 75점 (D-022 기준) ✅
- **보강 권고: "72점 → 목표 ?" 수치가 명시됐으면 더 좋음.** 최소 기준만 있고 목표 수치 없음
- "광고문 제거/완화"가 AI 판단인지 룰 기반인지 spec 없음 → 구현 전 정의 필요

### 🟢 builder
- 2-pass 말투 변환 = 처리 단계 증가 → 응답 시간 영향 측정 필요
- 단계 분리(광고문 제거 → 말투 변환)는 디버깅 용이성↑ — 아키텍처 방향 맞음
- "광고문 제거/완화" 단계 = 규칙 기반이면 테스트 용이, AI 판단이면 일관성↓

### 🔴 growth
- 상품글 품질↑ → 말투 점수↑ → 발행 통과 → 쿠팡 파트너스 전환율↑ = 직접 수익 기여 경로 명확
- 말투 점수 개선 + 어필리에이트 클릭률로 이중 추적 권장

**최종 판정: ✅ GO 유지**
- 데이터 기반 결정, 통과 기준 있음
- "광고문 제거" 단계 spec 문서화 1건만 추가 권고
